# XGBoost Live Two-Way Algorithmic Trading Assistant

An end-to-end machine learning-based trading decision support system that combines traditional technical analysis with predictive modeling. The system detects EMA 9/20 crossover events, evaluates market conditions using engineered technical features, and predicts the probability of a trade reaching its profit target before its stop loss.

The project demonstrates a complete machine learning workflow:

- Financial data collection through APIs
- Technical indicator computation
- Trade labeling and dataset generation
- Feature engineering
- Model training and comparison
- Probability threshold optimization
- Strategy backtesting
- Interactive Streamlit dashboard deployment

## Project Objective

Traditional EMA crossover strategies often generate false signals during sideways or highly volatile market conditions.

This project introduces a Machine Learning quality-control layer that evaluates each EMA crossover before execution.

Instead of blindly taking every crossover signal, the model estimates the probability that the setup will succeed based on market structure, momentum, volatility, and volume characteristics.

---

## Trading Strategy

The core trading logic is based on:

- EMA 9
- EMA 20
- EMA 9/20 Bullish Crossovers
- EMA 9/20 Bearish Crossovers
- RSI (Relative Strength Index)
- ATR (Average True Range)

### Bullish Setup

EMA 9 crosses above EMA 20

→ Potential Buy Opportunity

### Bearish Setup

EMA 9 crosses below EMA 20

→ Potential Sell Opportunity

Each detected setup is evaluated by the machine learning model before execution.

---

## Dataset Generation Process

Historical market data is downloaded using Yahoo Finance.

The system:

1. Detects EMA crossover events.
2. Records market conditions at the crossover.
3. Calculates ATR-based profit and stop-loss levels.
4. Monitors future candles for up to 24 hours.
5. Assigns labels:

### Win (1)

Take Profit hit before Stop Loss.

### Loss (0)

Stop Loss hit before Take Profit.

This converts raw market data into a supervised machine learning dataset.

## Model Benchmark Results

| Model | Accuracy | ROC-AUC | Win Recall | F1 Score |
|---------|---------|---------|---------|---------|
| XGBoost | 57.1% | 0.574 | 0.459 | 0.386 |
| Random Forest | 51.6% | 0.513 | 0.514 | 0.384 |
| Logistic Regression | 53.2% | 0.510 | 0.432 | 0.352 |

Based on out-of-sample forward testing, XGBoost achieved the strongest overall predictive performance and was selected as the production model.

## Probability Threshold Optimization

Instead of using the default 50% probability cutoff, the project optimizes the execution threshold using Precision-Recall analysis.

This allows the strategy to better balance:

- Trade Frequency
- Win Rate
- Risk Management
- Risk-Reward Efficiency

The threshold optimizer identifies the mathematically optimal probability threshold for trade execution.

Example optimization output:

- Optimal Threshold: 0.4093
- Maximum F1 Score: 0.4839
- Precision: 0.3448
- Recall: 0.8108

### Module Overview

Downloads historical OHLCV market data from Yahoo Finance.

#### indicators.py

Calculates technical indicators including:

- EMA 9
- EMA 20
- EMA 50
- RSI
- ATR

Detects EMA crossover events and labels each trade as a Win or Loss based on future price action.

Runs the complete data pipeline and generates:
